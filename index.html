什么是JT？

为web开发者提供开发javascript辅助及半自动化工具集。

具有以下功能与特性：

  • 代理服务
  • 项目构建
  • 极限压缩
  • 插件支持

快速入门

安装

jt已经发布在npm上，通过以下命令进行安装

npm install -g jt-cli

注意：请一定加上-g参数

安装后允许你在任意目录下运行JT

JT如何工作？

初始化配置文件

JT的运行依赖于相应的文件内的配置规则，在运行JT之前需要编写好配置文件，JT已经提供了一套默认配置，可以方便快捷地生成。

jt init

建议在项目文件夹根目录下执行。JT执行时按当前目录向上逐级查找配置文件，在项目根目录生成配置文件可以更加方便JT查找配置文件。

命令执行后会在当前路径下生成.jt文件夹

├──┬.jt
   ├────config.js
   ├────fs.js
   ├────project.js
   └────server.js

友好命令行界面

执行 jt -h 查看相关帮助

  Usage: jt [options] <arguments>

  Option:

配置文件

config.js

var config = {
    base: '../', 
    fs: {
        list: require('./fs.js'),
        ignorePath: []
    },
    project: require('./project.js'),
    server: require('./server.js'),
    compressor: {
        gzip: true
    }
};

base 指定文件夹路径，配置文件内所有相对路径都以该目录作为相对目录。这里推荐指定存放静态文件的目录为base路径。

compressor.gzip 压缩器在压缩文件时是否按gzip格式压缩判断

fs.js

var fileSystem = {
    'folder': {
        'file': [
            'file', {
                'compressor': 'string',
                'value': 'string'
            }
        ]
    }
};

project.js

var project = {
    'project': {
        'description': 'project\' description',
        'files': [
            'fileA',
            'fileB'
        ]
    }
};

server.js

var server = {
    slowSpeedSimulate: {
        enable: false,
        highWaterMark: 100,
        interval: 10,
    },
    port: 8080,
    description: 'default',
    list: {},
    hosts: {}
};
